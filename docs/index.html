Zonzabone

Zonzabone is a javascript framework which is tailored to satisfy the needs of zadar/zonza+. Instead of writing an essay about the beauty of it, it might better to create a short list of the features and characteristics.

  1. Domain specific - Zonzabone was created to support one particular product.
  2. Mid level - most of the mature frameworks (e.g. Backbone, Angular) have something for developers of any level of skill. With Zonzabone you have to be a capable javascript developer to utilise the library effectively.
  3. Traditional MVC - due to the domain-specificity, Zonzabone is great with traditional MVC. However this means that Zonzabone is only of help if you want to create controller heavy apps/miniapp/widgets. Apart from that you can be very flexible with architecture. Like:
    • controller with one view
    • controller with a collection of views
    • controller with one view which has sub-views
    • controller with sub-controllers and individual views
    • etc
  4. Not so traditional MVC - Ok this was under the belt. The truth is the views are able to render templates. So the actual design pattern is more like MVCVM
  5. Backbone inspired - you can see some Backbone-inspired features are implemented, but besides those, Zonzabone might be quite different. It lacks the 'ease' of Backbone, and great documentation (unfortunately) but most importantly you will never end up writing heavy, smart-head views. Hopefully...
  6. Spooky - 95% of js apps out there use constructor functions and try to mimic classes to implement characteristics of object-oriented programming. Not like Zonzabone. The actual implementation is a customized, mixed version of the functional and prototypal inheritance pattern published by Douglas Crockford in Good Parts. Thanks Boss.
  7. Cool
