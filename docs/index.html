Zonzabone

Zonzabone is a javascript framework which is tailored to satisfy the needs of zadar/zonza+. Instead of writing an essay about the beauty of it I think it's better to create a short list of the features and characteristics.

  1. Domain specific. It was created to support one particular product.
  2. Mid level. Most of the mature frameworks has something for developers on any level. With Zonzabone you have to be a medium level genuine javascript developer to use it effectively.
  3. Traditional MVC Because of it's domain-specificness and tailoredness Zonzabone is great with traditional MVC. However this only means that Zonzabone is a big help if you want to create controller heavy apps/miniapp/widgets. Apart from that you can be very flexible with architecture. Like:
    • controller with one view
    • controller with a collection of views
    • controller with one view which has sub-views
    • controller with sub-controllers and each of them has a view
    • etc
  4. No so traditional MVC Ok this was under the belt. The truth is the views are able to render templates. So the real design patterns is like MVCVM
  5. Backbone inspired As you can see some of Backbone feature are implemented. But besides them Zonzabone might be quite different. Like it lacks of the "easiness" of Backbone. And the great documentation unfortunately. But most importantly you will never end up writing heavy, smart-head views. Hopefully...
  6. Spooky 95% of js apps out there use constructor functions and try to mimic classes to implement characteristics of object-oriented programming. Not like Zonzabone. The actual implementation is a customized, cross-overed version of the functional and prototypal inheritance pattern published by Douglas Crockford in Good Parts. Thanks Boss.
  7. Cool
